Add (append) an index entry from a file in disk.

A new entry will always be inserted into the index; if the index already contains an entry for such path, the old entry will **not** be replaced.

The file `path` must be relative to the repository's working folder and must be readable.

This method will fail in bare index instances.

int git_index_open_bare ( git_index **  index,
const char *  index_path 
)

Create a new Git index object as a memory representation of the Git index file in 'index_path', without a repository to back it.

Since there is no ODB behind this index, any Index methods which rely on the ODB (e.g. index_add) will fail with the GIT_EBAREINDEX error code.

Parameters:
index the pointer for the new index
index_path the path to the index file in disk
Returns:
0 on success; error code otherwise
