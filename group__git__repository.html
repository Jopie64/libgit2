void git_repository_free ( git_repository *  repo  ) 

Free a previously allocated repository.

Note that after a repository is free'd, all the objects it has spawned will still exist until they are manually closed by the user with `git_object_close`, but accessing any of the attributes of an object without a backing repository will result in undefined behavior

int git_repository_open ( git_repository **  repository,
const char *  path 
)

Open a git repository.

The 'path' argument must point to an existing git repository folder, e.g.

/path/to/my_repo/.git/ (normal repository) objects/ index HEAD

/path/to/bare_repo/ (bare repository) objects/ index HEAD

The method will automatically detect if 'path' is a normal or bare repository or fail is 'path' is neither.

Parameters:
repository pointer to the repo which will be opened
path the path to the repository
Returns:
0 on success; error code otherwise

Open a git repository by manually specifying all its paths.

Parameters:
repository pointer to the repo which will be opened
git_dir The full path to the repository folder e.g. a '.git' folder for live repos, any folder for bare Equivalent to $GIT_DIR. Cannot be NULL.
git_object_directory The full path to the ODB folder. the folder where all the loose and packed objects are stored Equivalent to $GIT_OBJECT_DIRECTORY. If NULL, "$GIT_DIR/objects/" is assumed.
git_index_file The full path to the index (dircache) file Equivalent to $GIT_INDEX_FILE. If NULL, "$GIT_DIR/index" is assumed.
git_work_tree The full path to the working tree of the repository, if the repository is not bare. Equivalent to $GIT_WORK_TREE. If NULL, the repository is assumed to be bare.
Returns:
0 on success; error code otherwise

Open a git repository by manually specifying its paths and the object database it will use.

Parameters:
repository pointer to the repo which will be opened
git_dir The full path to the repository folder e.g. a '.git' folder for live repos, any folder for bare Equivalent to $GIT_DIR. Cannot be NULL.
object_database A pointer to a git_odb created & initialized by the user (e.g. with custom backends). This object database will be owned by the repository and will be automatically free'd. It should not be manually free'd by the user, or this git_repository object will become invalid.
git_index_file The full path to the index (dircache) file Equivalent to $GIT_INDEX_FILE. If NULL, "$GIT_DIR/index" is assumed.
git_work_tree The full path to the working tree of the repository, if the repository is not bare. Equivalent to $GIT_WORK_TREE. If NULL, the repository is assumed to be bare.
Returns:
0 on success; error code otherwise
